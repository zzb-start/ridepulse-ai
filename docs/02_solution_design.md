# 技术方案设计

> 对应开题报告Part 2。本文档为扩展版，包含完整的架构说明和技术细节。

## 方案概述

RidePulse AI是一套由6个Agent组成的全球骑行用户需求雷达系统。它将碎片化用户声音转化为可追溯、可排序、可进入产品待办的需求证据卡。架构从第一天起按多语言、多地区设计——首轮MVP以中国平台24条真实数据验证全链路可行性。

## 六Agent流水线

1. **采集Agent** — 连接20+全球数据源，保留URL/时间/SKU/版本
2. **治理Agent** — MinHash+BGE-M3双重去重，语言识别，垃圾过滤
3. **理解Agent** — 轻量模型+JSON Schema → 分类标签
4. **发现Agent** — BGE-M3+HDBSCAN聚类+异常检测
5. **证据审校Agent** — 六维评分+高推理模型生成证据卡
6. **交付Agent** — 飞书多维表格+待办+实验闭环

## 核心创新

1. **证据链而非摘要** — 每条洞察强制引用原文ID/URL/时间
2. **跨平台校正** — 五类来源分层计权
3. **沉默需求发现** — 严重度独立于频次评分
4. **实验闭环** — 证据卡→飞书待办→验证回流

## MVP试点目标

| 指标 | 目标值 |
|------|--------|
| 处理效率 | 单千条工时减少≥70% |
| 发现时延 | 批处理<4h，高风险<15min |
| 采纳率 | Top-20被PM采纳≥70% |
| 可追溯率 | 100%洞察可回链 |
| 行动转化 | 30天内进入PRD≥30% |

> ⚠️ 以上均为MVP试点目标值，非已实现结果。
